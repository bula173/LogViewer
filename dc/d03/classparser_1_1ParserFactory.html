Detailed Description

Factory class for creating parser instances.

Provides centralized parser creation with support for:

  • Automatic parser selection based on file extension
  • Explicit parser type specification
  • Runtime registration of custom parsers
  • Extension to parser type mapping
Thread Safety
The factory uses a static map for parser creators. Registration should be done during application initialization before concurrent access begins.
Design Pattern
Implements the Factory Method pattern with additional registry functionality for extensibility.

Registers a custom parser creator for a file extension.

Allows runtime registration of parsers for custom file formats. The creator function will be called whenever a file with the specified extension needs parsing.

Parameters
extensionFile extension (e.g., ".log", ".txt")
creatorFunction that creates parser instances
Returns
Optional error if registration failed, std::nullopt on success
Example:

return std::make_unique<MyLogParser>();
});

// Handle registration error
util::Logger::Error("Registration failed: {}", error->message());
}
