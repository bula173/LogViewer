Detailed Description

High-performance, thread-safe container for managing large collections of LogEvent objects.

EventsContainer provides efficient storage and access to LogEvent collections, optimized for handling very large log files (potentially millions of events). The container supports random access, iteration, and provides methods for virtual list control integration.

Performance Characteristics
  • Random access: O(1)
  • Insertion at end: O(1) amortized
  • Memory usage: Optimized for large datasets
Thread Safety
Thread-safe: All public methods are protected by internal locks.
  • Multiple threads can read concurrently (shared_lock)
  • Write operations use exclusive locks (unique_lock)
  • Safe for concurrent access from parser threads and UI thread
Lock Strategy
Uses std::shared_mutex for reader-writer lock pattern:
  • Read operations (GetEvent, Size): shared_lock allows concurrent reads
  • Write operations (AddEvent, Clear): unique_lock ensures exclusive access

Merges events from another container, sorted by timestamp.

Events are merged in timestamp order. If a timestamp field is not found, events are appended at the end in their original order.

Parameters
otherContainer with events to merge
existingAliasIdentifier for events already in this container
newAliasIdentifier for events from the 'other' container
timestampFieldName of the field containing timestamp (e.g., "timestamp", "time")
Note
All existing events will have their source set to existingAlias
All events from 'other' will have their source set to newAlias

Merges events from another container, sorted by timestamp.

This method combines events from both containers and sorts them by the specified timestamp field. Events from this container get existingAlias, and events from the 'other' container get newAlias as their source. Events without a valid timestamp will be appended at the end.

Parameters
otherThe container to merge from
existingAliasThe source identifier for existing events
newAliasThe source identifier for new events
timestampFieldThe field name containing the timestamp

Thread-safe: Uses exclusive lock for modification.
