Detailed Description

Primary application window providing log viewing and analysis capabilities.

MainWindow serves as the central hub for the LogViewer application, integrating data parsing, event display, filtering, and search functionality into a cohesive user interface. It implements the Observer pattern to receive notifications from data parsers and updates the display accordingly.

Key Features
  • File opening with drag-and-drop support and recent files menu
  • Virtual list controls for efficient display of large datasets (millions of events)
  • Multi-criteria filtering with apply/clear functionality
  • Full-text search across event data with results panel
  • Resizable panels with persistent layout preferences
  • Theme support (light/dark modes)
  • Progress indication during file parsing
  • Debug data population for development and testing
Architecture
The window uses a hierarchical splitter-based layout:
+-----------------------------------------------------------+
| Menu Bar & Toolbar |
+-----------------------------------------------------------+
| Left Panel | Main Content |
| (Filters) | +-----------------------------------+ |
| | | Event List | |
| | | (Virtual List) | |
| | +-----------------------------------+ |
| | | Event Details | |
| | | (Right Panel) | |
+-----------------------------------------------------------+
| Search Results (Collapsible) |
+-----------------------------------------------------------+
| Status Bar |
+-----------------------------------------------------------+
Thread Safety
This class should only be accessed from the main GUI thread. Data parsing operations may run on background threads but UI updates are marshaled to the main thread through the observer pattern.
Performance Considerations
  • Uses virtual list controls to handle millions of events efficiently
  • Implements lazy loading and on-demand data retrieval
  • Batched event processing to minimize UI update overhead
  • Progress reporting to maintain UI responsiveness during large file parsing

overridevirtual

Called when a batch of new events is found during parsing.

Processes a batch of events efficiently, minimizing UI update overhead. This is the preferred method for high-throughput parsing scenarios.

Parameters
eventBatchVector of (id, items) pairs representing the events
Note
This method is more efficient than multiple NewEventFound calls
Filter controls are updated after batch processing is complete

Implements parser::IDataParserObserver.
