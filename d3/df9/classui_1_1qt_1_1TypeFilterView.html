Detailed Description

Qt-backed implementation of the log-event type filter panel.

Displays a scrollable, checkable list of event types (e.g. ERROR, INFO, WARN). The list is populated by ReplaceTypes() whenever a new log file is loaded. Checking/unchecking triggers the m_onFilterChanged callback, which drives the async filter pipeline in MainWindowPresenter.

Relation to ITypeFilterView
All public methods except SetCheckedTypes() are part of the ITypeFilterView toolkit-agnostic interface. SetCheckedTypes() is a Qt-specific extension used by FilterProfilesPanel to restore type selections from a saved profile.

◆ SetCheckedTypes()

void ui::qt::TypeFilterView::SetCheckedTypes ( const std::vector< std::string > &  types)

Restores a type selection from a saved filter profile.

Checks only the types whose string representation is present in types, unchecking everything else. Triggers no FilterChanged callback — the caller is responsible for re-applying the filter (e.g. via MainWindow::OnApplyFilterClicked()) after this call.

Parameters
typesType strings to check. An empty vector is treated as "check everything" (equivalent to SelectAll()).
