Detailed Description

Panel for saving and restoring named filter profiles.

Profiles capture the complete filter state (time range, actor checks, type filter) and persist them to a JSON file in the app config directory.

Responsibility split
The panel is deliberately kept thin. It knows nothing about the other filter panels. All cross-panel coordination is delegated to MainWindow via the two signals below.
Persistence
Profiles are stored in <appdata>/filter_profiles.json (one JSON array). The file is loaded in the constructor and saved after every mutating operation (save / delete).
Double-click shortcut
Double-clicking a list item is equivalent to clicking Load.

◆ StoreProfile()

void ui::qt::FilterProfilesPanel::StoreProfile ( const FilterProfile &  profile)

Stores (or replaces) a profile in the panel and persists it to disk.

Called by MainWindow after it has gathered state from all filter panels in response to the SaveRequested signal. If a profile with the same name already exists it is replaced in-place; otherwise the profile is appended.
