Detailed Description

Improved IModel with weak_ptr observers for automatic cleanup.

This interface replaces the raw pointer-based observer pattern with weak_ptr-based observers that automatically detect and remove destroyed views. It provides thread-safe observer notifications with reader-writer locking.

Thread Safety
All observer management and notification methods are thread-safe using shared_mutex for concurrent reads and exclusive writes.
Lifetime Management
  • Views register as shared_ptr, internally stored as weak_ptr
  • Destroyed views are automatically detected and cleaned up
  • No dangling pointers possible
  • Views control their own lifetime via shared_ptr reference counting
Migration Guide
Old API (deprecated):
model->RegisterOndDataUpdated(view.get()); // Raw pointer

New API (recommended):

auto view = std::make_shared<MyView>();
model->RegisterView(view); // Shared pointer - safe!

IModel for backward compatibility interface

◆ NotifyDataChanged()

void mvc::IModelObservable::NotifyDataChanged ( )
inlineprotected

Notifies all registered views that data has changed.

Iterates through all observers and calls their OnDataUpdated method. Dead observers (destroyed views) are automatically detected via weak_ptr and skipped without error.

Exception Safety
If a view's OnDataUpdated throws, the exception propagates after notifying remaining views. Uses best-effort notification.

void mvc::IModelObservable::RegisterView ( ViewPtr  view)
inline

Registers a view as observer with automatic cleanup.

The view is stored internally as weak_ptr so it's automatically removed when destroyed. No dangling pointers.

Parameters
viewShared pointer to view to register
Example
auto container = std::make_shared<EventsContainer>();
auto view = std::make_shared<ItemDetailsView>();
container->RegisterView(view); // Safe lifetime management
view.reset(); // View destroyed
container->NotifyDataChanged(); // Safe: weak_ptr detects deletion
