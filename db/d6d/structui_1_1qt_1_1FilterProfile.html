Detailed Description

Snapshot of all active filter state that can be named and persisted.

A profile captures the complete, user-visible filter state at a single point in time so that it can be restored later with a single click.

Covered filter dimensions

Field Restored by
timeRange TimeRangeFilterPanel::SetState() + Apply()/Clear()
uncheckedActors ActorsPanel::RestoreUncheckedActors()
checkedTypes TypeFilterView::SetCheckedTypes()

JSON schema

{
"name": "Production errors",
"timeRange": { "field": "timestamp", "from": "2026-01-01", "to": "", "active": true },
"uncheckedActors": ["MyDef\u0000ServerA"],
"hasTypeFilter": true,
"checkedTypes": ["ERROR", "FATAL"]
}
Note
Actor keys use the ActorKey::Encode() / ActorKey::Decode() helpers defined in ActorDefinition.hpp.
