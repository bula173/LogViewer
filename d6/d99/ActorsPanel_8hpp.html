Detailed Description

Panel showing per-actor event statistics and actor-based filtering.

Actors are extracted from log events using regexp-based ActorDefinition rules. The panel displays a tree where each top-level row represents one definition and child rows represent the individual matched actors (in capture-group mode).

Interaction with filter profiles

The check state of every actor row can be saved and restored as part of a FilterProfile via GetUncheckedActors() and RestoreUncheckedActors(). Actor keys use the ActorKey::Encode() / ActorKey::Decode() helpers from ActorDefinition.hpp.
